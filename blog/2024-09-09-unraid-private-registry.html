Installing and confurering private Docker registry

This is a guide on how to install and configure a private Docker registry on Unraid. A private Docker registry is a great way to store your Docker images and share them with your team. It is also a great way to speed up your builds by caching your images locally.

DockerRegistry is a great tool for this. You can find the documentation here (DockerRegistry)

Select vairable from the Config Type: dropdown

Add the following variables:

Name: REGISTRY_AUTH_HTPASSWD_PATH: Key: REGISTRY_AUTH_HTPASSWD_PATH Value: /auth/htpasswd

Name: REGISTRY_HTTP_TLS_CERTIFICATE: Key: REGISTRY_HTTP_TLS_CERTIFICATE Value: /certs/certificate.crt

Name: REGISTRY_HTTP_TLS_KEY: Key: REGISTRY_HTTP_TLS_KEY Value: /certs/private.key

Select path from the Config Type: dropdown

Add the following paths:

Name: auth: Container Path: /auth Value: /mnt/user/appdata/registry/auth

Name: certs: Container Path: /certs Value: /mnt/user/appdata/registry/certs

This will create a private Docker registry with basic authentication. You can now push and pull images from your registry.

On the container we will need to make to following changes:

cd /mnt/user/appdata/registry mkdir auth mkdir certs docker run --entrypoint htpasswd httpd:2 -Bbn [user_name] [password] > auth/htpasswd

sudo apt-get update sudo apt-get install openssl openssl genrsa -out private.key 2048 echo " [ req ] default_bits = 2048 distinguished_name = req_distinguished_name req_extensions = req_ext x509_extensions = v3_ca # The default uses v3_ca for self-signed certs prompt = no [ req_distinguished_name ] C = NL ST = North Holland L = Amsterdam O = MyCompany CN = enter your domain or host ip name here [ req_ext ] subjectAltName = @alt_names [ v3_ca ] subjectAltName = @alt_names [ alt_names ] IP.1 = enter your domain or host ip name here DNS.1 = tower" > openssl.cnf openssl req -x509 -new -key private.key -out certificate.crt -days 365 -config openssl.cnf

This will create a self-signed certificate for your registry. You can now push and pull images from your registry.

Copy the certificate and key to the registry certfolder :

cp certificate.crt /mnt/user/appdata/registry/certs cp private.key /mnt/user/appdata/registry/certs

Restart the container and you should now be able to access your registry on https://[your-ip]:5000

If you are connecting from a remote machine you will need to add the certificate to your trusted certificates. scp certificate.crt remoteuser@remotehost:/etc/docker/certs.d/[docker-registry-ip]:5000/ca.crt sudo systemctl restart docker

You can now push and pull images from your registry.

If you are using a self-signed certificate you will need to add the certificate to your trusted certificates.

docker login [docker-registry-ip]:5000 docker tag [docker-registry-ip]:5000/[image-name] docker push [docker-registry-ip]:5000/[image-name] curl -X GET https://[docker-registry-ip]:5000/v2/_catalog
